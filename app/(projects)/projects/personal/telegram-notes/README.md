## Why
Free-form personal notes and reminders in Telegram through spoken hook phrases (the Hook entity, step 187). A single entry point for the spoken command mode: the bot receives messages, recognizes a registered hook from the global project_hooks table by the first words, and routes the message to the owning project. Three hooks: "remember this" saves to memory, "remind me …" sets a date-based reminder, "what did I save about …" searches memory and answers in the chat.

## Efficiency
Notes stop getting lost in conversations: one spoken phrase instead of keeping a database by hand. Search is semantic (LightRAG vector similarity), not exact word matching. Reception via fractera-cron polling (getUpdates) needs NO webhook/HTTPS — it works in IP mode and, crucially, WITHOUT the native Hermes gateway (zero runtime load on Hermes, full self-sufficiency). A cheap model via OPENAI_API_KEY does the summary and classification — very low token cost.

## Reuse
The fetch/detect-hook/reply nodes are reusable by any future substrate Telegram bot; hook detection over the global project_hooks table is the shared entry point of ALL spoken automations (not just notes); the "cron-poll getUpdates instead of webhook" pattern applies to every integration without HTTPS; the last_update_id cursor in SCHEMA is a template for cursor persistence in any poller.

## Result
A working personal bot @fractera_bot: within a minute, a hook phrase in Telegram becomes a memory record, a date-based reminder, or an answer drawn from memory. A standalone project page /projects/personal/telegram-notes with a description, a process canvas diagram, a run panel, a runs dashboard, a cron queue, and a results table (summaries of all records with server-side search).
