## Why
Личные заметки и напоминания в Telegram в свободной форме через произнесённые фразы-хуки (сущность Hook, шаг 187). Единая точка входа голосового командного режима: бот принимает сообщения, по первым словам распознаёт зарегистрированный хук из глобальной таблицы project_hooks и маршрутизирует сообщение в проект-владелец. Три хука: «запомни это» сохраняет в память, «напомни мне …» ставит напоминание по дате, «что я сохранял про …» ищет по памяти и отвечает в чат.

## Efficiency
Заметки перестают теряться в переписке: одна произнесённая фраза вместо ручного ведения базы. Поиск семантический (векторная близость LightRAG), а не по точному совпадению слов. Приём через поллинг fractera-cron (getUpdates) НЕ требует webhook/HTTPS — работает в IP-режиме и, главное, БЕЗ нативного шлюза Hermes (нулевая рантайм-нагрузка на Hermes, самодостаточность). Summary и классификацию делает дешёвая модель по OPENAI_API_KEY — очень низкий расход токенов.

## Reuse
Узлы fetch/detect-hook/reply переиспользуемы любым будущим Telegram-ботом субстрата; детект хука по глобальной project_hooks — общий вход ВСЕХ голосовых автоматизаций (не только заметок); паттерн «cron-поллинг getUpdates вместо webhook» применим ко всем интеграциям без HTTPS; курсор last_update_id в SCHEMA — образец персиста курсора для любых поллеров.

Работающий личный бот @fractera_bot: фраза-хук в Telegram в течение минуты превращается в запись в памяти, в напоминание по дате или в ответ из памяти.
